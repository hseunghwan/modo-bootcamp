# 다국어 노래방 자막 영상 SaaS — 가치사슬(Value Chain) 분석

## 개요
본 문서는 다국어 싱얼롱(노래방) 자막 자동화 SaaS의 가치 창출 구조를 분석하여, 비용·품질·속도·합법성 측면에서의 개선 지점을 도출한다.

## 1) 기본활동(Primary Activities)
### a. 콘텐츠 수급·입력
- 입력: 오디오/영상 파일, 가사 텍스트(원어), 메타데이터(언어/가수/BPM/키), 배경/브랜딩 자산
- 이슈: 가사/음원 권리 확인, 포맷 다양성, 대용량 업로드 안정성(Resumable)
- 개선: 업로더 SDK·링크 인제스트(YouTube/S3), 포맷 자동감지, 권리 체크리스트(Pre-flight)

### b. 처리(분석·정렬·번역·타이밍)
- ASR/가사 정렬(음절/박자), 언어감지, MT(의역/금칙어 처리), 타임코드 생성, 줄바꿈/분절, 이펙트 큐 생성
- 이슈: 비라틴 문자·RTL 언어·고유명사 처리, 듀엣/콜&리스폰스, 라이브 음원 노이즈
- 개선: 카라오케 특화 정렬엔진, 도메인 MT 사전/용어집, 품질 지표(QoS)와 휴리스틱 후처리

### c. 렌더링·인코딩·검수
- 싱얼롱 하이라이트, 진행 바, 색상/폰트/자간, 듀엣 분할, 자막 오버레이/박자 이펙트
- 이슈: 저지연 대량 렌더, 코덱/비트레이트 규격, 플랫폼 탐지(저작권/부적합) 회피 설계
- 개선: GPU 병렬 렌더 파이프라인, 프리셋(YouTube/TikTok/Reels), 자동 품질검사(클리핑/위치/가독성)

### d. 배포·채널 연동
- 산출물: 동영상 파일, SRT/ASS, 썸네일/챕터/메타
- 이슈: API/웹훅, 플랫폼별 제한(길이/해상도), 저작권 자동탐지 대응
- 개선: 채널 연동 커넥터, 예약 게시/버전 관리, 자동 재인코딩/자동자막 동기화

### e. 서비스·지원
- 온보딩 튜토리얼, 가이드라인(권리/번역/렌더), 품질 피드백 루프, SLA·모니터링
- 이슈: 스트라이크 대응, 분류·검색, 대량 작업 실패 처리
- 개선: 권리 리스크 헬스체크, 배치 큐 재시도/롤백, 품질 리뷰 어노테이션

## 2) 지원활동(Support Activities)
### a. 기술개발(R&D)
- 카라오케 정렬(음절/박자), 멀티랭 번역 품질(의역·운율·금칙어), 폰트/스크립트 렌더 엔진, 성능 최적화
### b. 인프라
- 클라우드 GPU/스토리지/미디어 서비스, 로그/트레이싱/비용 관제, 데이터 거버넌스
### c. 인적자원
- 음성/자연어/미디어 처리 엔지니어, 권리/법무, 크리에이티브 모션 디자이너, 테크니컬 CSM
### d. 조달
- 폰트/사운드 이펙트 라이선스, 번역 사전/용어집, 테스트 카탈로그(다언어/장르 다양)

## 3) 비용·품질·속도 관점의 레버리지 포인트
- 비용: GPU 렌더/인코딩 병렬 최적화, 스팟 인스턴스/예약/오토스케일, 캐시·증분 렌더
- 품질: 정렬 정확도(음절 단위), 번역 가독성(의역/욕설/금칙어), 가시성(폰트/대비/배치), 스테레오타입/문화적 민감성 필터
- 속도: 배치 큐/샤딩, 스트리밍 파이프라인, 선계산(비트/구간), 프리셋 템플릿

## 4) 경쟁우위 제안
1) “합법성 내장형” 파이프라인: 업로드 단계 권리·탐지 체크, 가사 DB 매칭, 리스크 경고/차단.
2) “싱크 품질 모듈”로 차별화: 박자·음절 레벨 정밀도와 QoS 대시보드 공개.
3) “플랫폼 프리셋” 제공: YouTube/TikTok/Reels 목적지 최적화를 버튼 한 번으로.
4) “대량 배치·API” 강화: 채널/플랫폼 파트너의 운영 워크플로에 맞춘 확장성.
