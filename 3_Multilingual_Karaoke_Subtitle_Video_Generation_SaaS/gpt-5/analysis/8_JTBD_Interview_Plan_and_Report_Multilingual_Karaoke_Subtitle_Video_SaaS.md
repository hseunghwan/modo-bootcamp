# 다국어 노래방 자막 영상 SaaS — JTBD 심층 인터뷰 계획 & 결과 보고 템플릿

## 1) 목적
- 고객이 “노래를 함께 부를 수 있는 영상(싱얼롱)”을 만들 때 실제로 ‘무슨 일을’ 해내려 하는지(Progress)를 규명
- 구매·해지·대체의사 결정 요인(품질·합법성·속도·운영 통합)을 정량/정성으로 확인

## 2) 리크루팅 가이드(스크리너)
- 최근 3개월 내 싱얼롱/자막 영상 제작 경험
- 다국어 제작/배포 경험(있음/계획)
- 권리/스트라이크 이슈 경험 여부
- 월 제작량(볼륨), 현재 사용 툴/외주 방식

## 3) 인터뷰 플로우(60분)
1) 맥락 파악(10’) — 최근 제작 프로젝트, 목표, 산출물
2) 프로세스 재현(15’) — 입력→정렬/번역→렌더→게시 단계별 도구/시간/문제
3) Pain·Goal 딥다이브(15’) — 실패/지연/스트라이크 사례, 비용/품질 트레이드오프
4) 대안 비교(10’) — 기존 툴/외주 선택 이유, 전환/해지 계기
5) MVP 반응(10’) — 정밀 싱크·합법성·프리셋·배치/API 데모 피드백

## 4) 핵심 질문 예시
- “음절/박자 싱크가 어긋나면 어떤 일이 생기나요? 수정 시간/방법은?”
- “다국어 번역에서 가장 어려운 점(의역/금칙어/문화적 표현)은?”
- “스트라이크/권리 이슈를 어떻게 예방/대응하시나요? 무엇이 불안 요소인가요?”
- “대량 제작 시 병목은 어디인가요? 배치/API가 있다면 어떤 작업을 줄일 수 있나요?”
- “구매 결정을 바꿨던 순간(Trigger)과 해지/대체를 고민했던 순간은?”

## 5) 수집 지표(예시)
- 단계별 시간(TAT), 실패율, 재작업 빈도
- 품질 체감 지표: 싱크 정확도, 가독성, 번역 만족도(리커트)
- 합법성: 사전 체크 여부, 스트라이크 경험, 정책 이해도
- 운영: 배치 크기, API 필요 기능, 프리셋 적합률

## 6) 코딩 프레임(분류 체계)
- JTBD: “전 세계 시청자와 함께 부를 수 있는 영상을 빠르고 안전하게 만든다”
- Pain: 정렬/번역/권리/렌더/배포
- Outcome: 시간 단축, 스트라이크 0, 품질 일관성, 대량 자동화

## 7) 결과 보고서 템플릿
### 요약
- 표본(N), 핵심 인사이트 3~5개, MVP 개선점/우선순위
### 페르소나별 인사이트
- P1(채널), P2(교육), P3(레이블), P4(업장)… 각 상·하위 3개 Pain/Goal
### AOS·DOS 반영
- 인터뷰 점수를 AOS/DOS 표로 업데이트 → Q1 항목 재검증
### 아티팩트
- 인용(Quotes), 화면녹화, 결과물 비교(전/후), 워크플로 차트

## 8) 동의/윤리
- 익명 처리, 저작권/사생활 보호, 사용동의서 수집
